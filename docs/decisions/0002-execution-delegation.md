# ADR-0002: 실행 위임 모델

날짜: 2026-07-03
상태: 승인

## 맥락
유저 지시(2026-07-03)에 따라 에이전트 간 역할을 명확히 분리할 필요가 있었다. 이 프로젝트는 자격증명 노출·감염기기를 다루는 보안 도메인이라 dual-use로 오탐될 수 있는 안전조치(safety guardrail)가 실행 중 발동할 가능성이 있다.

## 결정
- 메인 에이전트(Fable)는 실행계획 수립·사고·산출물 검수만 담당하고 직접 코드/문서를 실행하지 않는다.
- 실제 실행은 서브에이전트에 위임: 복잡한 빌드·설계 판단이 필요한 작업은 opus, 기계적·반복적 작업은 sonnet.
- Fable이 안전가드에 걸리면 opus 등 다른 경로로 우회를 시도하지 않는다. 즉시 작업을 중단하고 유저에게 보고한다(유저가 필요 시 codex gpt5.5 xhigh로 직접 진행).

## 근거
보안 도메인에서는 안전조치가 정당한 방어 목적 작업을 오탐할 수 있다. 이때 판단 주체를 에이전트가 아닌 유저로 고정해야 오탐 시에도 우회 시도로 인한 위험(정책 위반, 신뢰 저하)을 피할 수 있다.

## 영향
- 이후 모든 실행 작업은 Fable이 직접 하지 않고 opus/sonnet 서브에이전트를 통해 위임한다.
- 안전가드 발동 시 작업 로그와 함께 유저에게 즉시 보고하는 것이 기본 동작이 된다.
