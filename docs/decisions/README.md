# ADR 색인 (Architecture Decision Records)

구조적 의사결정(리포지토리 구조·모듈 경계·기술 스택 등 되돌리기 비싼 결정)을 기록한다. 번호는 증가만 한다(삭제·재사용 금지).

## 템플릿
새 ADR 작성 시 아래 형식을 그대로 사용, 파일명은 `NNNN-짧은-제목.md`.

```markdown
# ADR-NNNN: 제목

날짜: YYYY-MM-DD
상태: 승인 | 제안 | 폐기(대체: ADR-NNNN)

## 맥락
왜 이 결정이 필요했는가.

## 결정
무엇을 하기로 했는가.

## 근거
왜 이 방식인가(대안 대비).

## 영향
이 결정이 바꾸는 것(구조·워크플로·후속 작업).
```
